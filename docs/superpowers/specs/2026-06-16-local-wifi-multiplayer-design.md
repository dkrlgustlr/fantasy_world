# 로컬 Wi-Fi 2인 멀티플레이 설계

## 목표

판타지 왕국 기본판을 기준으로, 두 명이 각자의 아이폰에서 같은 게임 방에 접속해 플레이할 수 있는 로컬 웹앱을 만든다. 초기 버전은 같은 Wi-Fi 안에서만 동작하며, 인터넷 배포나 외부 실시간 DB는 사용하지 않는다.

## 1차 버전 범위

1차 버전은 완성된 자동 점수 계산 게임이 아니라, 실제 카드 게임을 온라인 손패/턴 도구로 옮기는 것을 목표로 한다.

- 2인 접속
- 방 코드 입장
- 각자 손패 비공개
- 덱에서 카드 뽑기
- 공개 버린 카드에서 카드 가져오기
- 손패에서 카드 버리기
- 실제 룰처럼 `뽑기 -> 버리기` 순서 강제
- 원본 기본판 53장만 사용
- 카드는 우선 텍스트 카드로 표시
- 점수는 자동 계산하지 않고 수동 점수판처럼 기록
- 선택형 카드 효과는 플레이어가 직접 선택

자동 점수 계산, 카드 이미지, 확장 카드, 밸런스 분석은 1차 버전 이후 단계로 미룬다.

## 접근 방식

초기 멀티플레이는 로컬 PC가 서버 역할을 하고, 두 아이폰이 같은 Wi-Fi에서 그 서버에 접속하는 방식으로 구현한다.

- PC: 게임 서버 실행
- 아이폰 2대: 사파리에서 서버 주소 접속
- 서버 주소 예: `http://192.168.x.x:3000`
- 방 입장: 방 생성 후 4~6자리 초대코드 입력
- 인원: 초기 버전은 2인 고정

이 방식은 비용이 들지 않고, 서버 배포 없이 빠르게 실제 아이폰 플레이를 검증할 수 있다.

## 대안과 선택 이유

### 선택안: 로컬 Node.js 서버 + WebSocket

서버가 덱, 손패, 버린 카드, 턴 상태를 단일 권한으로 관리한다. 각 아이폰은 자기 행동을 서버에 보내고, 서버가 검증한 새 게임 상태를 두 기기에 동기화한다.

장점은 구현이 비교적 단순하고, 비공개 손패와 턴 검증을 서버에서 안정적으로 처리할 수 있다는 점이다. 이 프로젝트의 1차 목표에 가장 적합하다.

### 대안: Supabase/Firebase 기반 웹 배포

다른 장소에서도 접속할 수 있고, 별도 PC 서버 없이 운영할 수 있다. 대신 계정, 배포, 비용 한도, 보안 규칙을 함께 신경 써야 한다.

초기 테스트 단계에서는 과하다. 로컬판이 안정된 뒤 원격 멀티가 필요해지면 전환한다.

### 대안: 한 기기 공유 플레이

한 폰이나 PC 화면에서 둘이 번갈아 조작한다. 가장 쉽지만, 상대 손패를 숨기는 경험이 약해져 목표와 맞지 않는다.

## 게임 흐름

1. 한 플레이어가 PC에서 서버를 실행한다.
2. 두 아이폰이 같은 Wi-Fi에 접속한다.
3. 첫 번째 플레이어가 웹앱에서 방을 만든다.
4. 서버가 초대코드를 생성한다.
5. 두 번째 플레이어가 초대코드를 입력해 입장한다.
6. 두 명이 준비 완료를 누르면 서버가 덱을 섞고 각자 7장씩 나눠준다.
7. 현재 턴 플레이어는 덱 맨 위 카드 또는 공개 버린 카드 중 하나를 가져온다.
8. 손패에서 카드 1장을 버린다.
9. 공개 버린 카드가 10장이 되면 게임이 종료된다.
10. 각 플레이어가 수동 점수판에 점수를 기록하고 결과를 확인한다.

## 화면 구성

### 입장 화면

- 방 만들기
- 초대코드 입력
- 플레이어 이름 입력

### 게임 화면

- 내 손패 7장
- 덱 영역
- 공개 버린 카드 영역
- 현재 턴 표시
- 선택한 카드 상세 설명
- 가져오기, 버리기 버튼

상대 손패는 카드 수만 보여주고 내용은 숨긴다.

### 종료 화면

- 내 최종 점수
- 상대 최종 점수
- 카드별 기본 힘, 보너스/페널티, 합계
- 승자 표시

1차 버전에서는 종료 화면이 자동 계산 결과를 만들지 않는다. 대신 플레이어가 카드별 점수를 직접 입력하고, 앱은 입력된 값의 합계만 계산한다.

## 서버 상태 모델

서버는 방마다 아래 상태를 가진다.

- 방 코드
- 플레이어 목록
- 덱
- 공개 버린 카드
- 각 플레이어 손패
- 현재 턴 플레이어
- 게임 단계: 대기, 진행, 종료
- 행동 로그

초기 버전은 서버 메모리에만 상태를 저장한다. 서버를 끄면 진행 중인 방은 사라진다.

## 카드 데이터

카드 데이터는 JSON으로 분리한다.

- 카드 ID
- 카드명
- 종류
- 기본 힘
- 설명 문구
- 효과 정의
- 이미지 경로

기본판 기준은 `fantasy_world_summary.md`와 `references/confirmed-cards.md`를 우선 근거로 삼는다.

1차 버전에서는 원본 기본판 53장만 등록한다. 카드는 실제 이미지 대신 종류별 색상과 텍스트로 표시한다. 이미지 경로 필드는 나중에 카드 이미지를 붙일 수 있도록 남겨두되, 1차 구현에서는 필수로 사용하지 않는다.

## 점수 계산

1차 버전은 자동 점수 계산을 구현하지 않는다. 앱은 점수판 역할만 한다.

- 각 플레이어는 게임 종료 후 카드별 기본 힘, 보너스/페널티, 합계를 직접 입력한다.
- 앱은 입력값을 더해 총점과 승자를 표시한다.
- 카드 효과 해석은 플레이어가 직접 처리한다.

나중에 자동 점수 계산을 추가할 때는 서버에서 수행한다. 서버가 계산해야 두 플레이어가 같은 결과를 볼 수 있고, 자동 밸런스 테스트를 붙이기 쉽다.

향후 자동 계산 순서는 다음을 기준으로 한다.

1. 와일드/복사/종류 변경 선택지를 플레이어가 직접 고르거나 앱이 추천한다.
2. 무효화 효과를 적용한다.
3. 페널티 제거 효과를 적용한다.
4. 기본 힘을 합산한다.
5. 보너스와 페널티를 합산한다.
6. 카드별 계산 내역과 총점을 반환한다.

복사 카드와 선택형 카드는 1차 버전에서 플레이어가 직접 판단한다. 자동 최적 선택은 후속 기능으로 둔다.

## 오류 처리

- 방 코드가 없으면 입장 실패 메시지를 보여준다.
- 방이 가득 찼으면 입장을 막는다.
- 자기 턴이 아니면 행동을 거부한다.
- 손패가 8장이 아닌 상태에서 버리기를 막는다.
- 카드를 뽑기 전에는 버리기를 막는다.
- 카드를 뽑은 뒤에는 버리기 전까지 추가로 뽑을 수 없다.
- 연결이 끊기면 재접속 안내를 보여준다.

초기 버전은 같은 브라우저에서 새로고침하면 같은 플레이어로 재접속할 수 있도록 로컬 저장소에 플레이어 토큰을 저장한다.

## 테스트 기준

- 같은 Wi-Fi에 연결된 아이폰 2대가 방 코드로 같은 방에 입장할 수 있다.
- 각자 자기 손패만 볼 수 있다.
- 한 명의 행동이 다른 기기에 반영된다.
- 턴이 아닌 플레이어는 카드를 가져오거나 버릴 수 없다.
- 턴 플레이어는 반드시 카드 1장을 가져온 뒤 카드 1장을 버려야 한다.
- 공개 버린 카드가 10장이 되면 게임이 종료된다.
- 종료 후 플레이어가 입력한 점수 합계와 승자가 양쪽에 동일하게 표시된다.

## 제외 범위

초기 버전에서는 아래 기능을 제외한다.

- 인터넷 원격 멀티플레이
- 계정/로그인
- 전적 저장
- AI 상대
- 3인 이상 플레이
- 앱스토어 배포
- 자동 점수 계산
- 카드 이미지 표시
- 선택형 카드 자동 최적화
- 카드 확장팩 밸런스 자동 분석

이 기능들은 기본 2인 로컬 플레이가 안정된 뒤 추가한다.
