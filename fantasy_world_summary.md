# 판타지월드(판타지 왕국 스타일) 기획/구현 정리

> 작업명은 사용자가 말한 **판타지월드**로 정리함.  
> 기반 참고 게임은 사진으로 확인한 **판타지 왕국(Fantasy Realms)** 스타일의 카드 조합 점수 게임이다.  
> 이 문서는 개인 로컬 게임 구현을 위한 기획/데이터 설계용 정리본이다.

---

## 1. 게임 핵심 구조

### 기본 컨셉

- 플레이어는 손패 카드 조합으로 점수를 만든다.
- 각 카드는 다음 요소를 가진다.
  - 카드명
  - 문양/종류
  - 기본 점수
  - 보너스
  - 페널티
  - 무효화 효과
  - 특수 처리 효과
- 최종 점수는 **기본 점수 + 보너스 + 페널티**로 계산한다.
- 일부 카드는 다른 카드를 **무효화**한다.
- 일부 카드는 카드의 문양/이름/점수 등을 바꾸거나 복사한다.

### 기본 진행 방식

1. 각 플레이어는 카드 7장을 손패로 받는다.
2. 자기 차례에 카드 1장을 가져온다.
   - 덱 맨 위 카드
   - 공개된 버린 카드 중 1장
3. 카드 1장을 버린다.
4. 공개 버린 카드 영역에 카드가 10장이 되면 게임 종료.
5. 각자 손패 점수를 계산하고 가장 높은 사람이 승리.

## 2. 구현 목표

처음부터 완성 게임을 만들기보다 아래 순서로 개발하는 것을 추천한다.

### 1단계: 점수 계산기

- 손패 7장을 선택한다.
- 카드별 점수를 자동 계산한다.
- 보너스/페널티/무효화 내역을 보여준다.

### 2단계: 최적 조합 추천

- 와일드 카드, 복사 카드, 문양 변경 카드의 가능한 선택지를 계산한다.
- 가장 점수가 높은 조합을 추천한다.

### 3단계: 실제 게임 진행

- 덱 셔플
- 플레이어 손패
- 버린 카드 영역
- 턴 진행
- 종료 조건
- 최종 점수판

### 4단계: UI 개선

- 카드 이미지 추가
- 카드 클릭 시 능력 설명 보기
- 무효화된 카드 흐리게 표시
- 점수 계산 내역 펼쳐보기

## 3. 카드 데이터 구조

카드는 JSON 형태로 관리하는 것이 좋다.

```json
{
  "id": "dragon",
  "name": "용",
  "suit": "야수",
  "base": 30,
  "text": "마법사가 없으면 -40점.",
  "effects": [
    {
      "type": "penalty_unless_has_suit",
      "suit": "마법사",
      "points": -40
    }
  ]
}
```

### 카드 데이터 필드

| 필드 | 설명 |
|---|---|
| id | 코드에서 사용할 고유 ID |
| name | 카드 이름 |
| suit | 카드 문양/종류 |
| base | 기본 점수 |
| text | 카드 설명 텍스트 |
| effects | 점수 계산용 효과 목록 |
| tags | 특수 분류가 필요할 때 사용 |
| image | 카드 이미지 경로 |

## 6. 점수 계산 순서 초안

점수 계산은 다음 순서로 구현하는 것이 좋다.

1. 손패 카드 목록을 만든다.
2. 와일드/복사/문양 변경 카드의 선택지를 생성한다.
3. 각 선택지마다 임시 손패를 만든다.
4. 무효화 효과를 계산한다.
5. 페널티 제거 효과를 적용한다.
6. 기본 점수를 더한다.
7. 보너스를 더한다.
8. 페널티를 더한다.
9. 특수 보너스 카드들을 계산한다.
10. 가장 높은 점수 선택지를 반환한다.

## 7. 구현 난이도가 높은 카드

아래 카드들은 단순 조건문으로 처리하기 어렵다.

| 카드 | 어려운 이유 |
|---|---|
| 변화의 책 | 어떤 카드 문양을 바꿀지 선택해야 함 |
| 신기루 | 어떤 자연 계열 카드로 취급할지 선택해야 함 |
| 변신 능력자 | 어떤 인물/장비 계열 카드로 취급할지 선택해야 함 |
| 도플갱어 | 어떤 내 카드를 복사할지 선택해야 함 |
| 생명의 샘 | 어떤 카드 기본 점수를 추가할지 선택해야 함 |
| 섬 | 어떤 물/불 카드의 페널티를 제거할지 선택해야 함 |
| 강령술사 | 버린 카드 더미에서 어떤 카드를 추가할지 선택해야 함 |
| 질서의 보석 | 연속 기본점수 조합 계산 필요 |
| 세계수 | 무효화되지 않은 카드들의 문양 중복 여부 확인 |
| 들불 | 무효화 예외가 중요함 |
| 대홍수 | 무효화 예외가 중요함 |
| 바실리스크 | 무효화 범위가 넓음 |
| 동굴 | 날씨 전체 무효화 |
| 폭우 | 불 전체 무효화 |
| 눈보라 | 물 무효화 + 여러 문양 페널티 |

## 8. 새 카드/확장 기획 시 밸런스 기준

처음부터 카드 수를 많이 늘리면 밸런스가 깨질 가능성이 높다.  
처음에는 기본 구조를 유지하고, 나중에 확장팩처럼 추가하는 것을 추천한다.

### 기본 추천 구조

| 구성 | 수 |
|---|---:|
| 문양 | 10개 |
| 각 문양 카드 | 5장 |
| 와일드/특수 | 3장 |
| 총 카드 | 53장 |
| 손패 | 7장 |

### 확장 추천 구조

- 기본 53장 완성 후 확장 카드 10~12장 추가
- 매 게임 확장 카드 전체를 넣지 말고 일부만 섞기
- 예: 확장 12장 중 무작위 6장만 사용

## 9. 카드 능력 패턴 예시

새 카드 만들 때는 아래 패턴을 섞으면 된다.

| 유형 | 예시 |
|---|---|
| 문양 보너스 | 야수 1장당 +8점 |

## 10. 점수 밸런스 기준

| 손패 상태 | 목표 점수 |

| 평범한 손패 | 80~120 |
| 잘 맞은 손패 | 130~180 |
| 강한 손패 | 180~250 |
| 대박 콤보 | 250~300+ |

| 카드 타입 | 권장 점수 |
|---|---:|
| 조건 없는 낮은 카드 | 0~10 |
| 쉬운 조건 보너스 카드 | 10~25 |
| 특정 카드 필요 카드 | 20~45 |
| 큰 페널티 있는 고점 카드 | 30~50 |
| 전용 콤보 완성 카드 | 40~80 |
| 매우 어려운 로망 콤보 | 100+ 가능 |

## 11. 로컬 구현 시 저작권 주의

개인 로컬용으로만 사용할 경우 실물 카드 사진을 참고하는 것은 현실적으로 문제 가능성이 낮다.  
하지만 공개 배포는 주의해야 한다.

주의할 점:

- 카드 이미지 공개 저장소 업로드 금지
- 카드 텍스트 전체 공개 배포 금지
- 공식 게임처럼 보이는 이름/로고 사용 금지
- 웹 공개, 앱스토어, 스팀 배포 시 저작권/상표권 문제 가능
- 공개 배포용이면 테마, 카드명, 카드 텍스트를 새로 만드는 것이 안전
